# docs/superpowers/ — 特性设计存档

本目录保存 Argos 各特性的**带日期设计记录**，分两类：

- **`specs/`** — 设计规格（命名格式：`YYYY-MM-DD-<name>-design.md`）
- **`plans/`** — 实现计划（命名格式：`YYYY-MM-DD-<name>.md`）

## 重要说明

这些文件是**某一时间点的历史快照**，不是持续维护的当前文档。
内容反映写作当时的设计决策，后续实现可能已经偏离。

**当前以用户为目标的特性文档请查阅 `docs/*.md`**，由那里的文档按需链接到此处的设计记录，而非反向引用。

## 目录惯例

- 每个特性建一个规格文件（`specs/`）和一个计划文件（`plans/`），以同一日期和特性名称命名。
- 文件名中的日期代表记录创建时间，不代表特性上线时间。
- 不要在此目录中编写"最新状态"或"当前架构"类内容——那类内容属于 `docs/`。
