# Argos — 产品定义(v5,基于实测证据)

> 一句话:**让用不起/用不上强模型的人,在「结果能被验证的事」上,用便宜模型也拿到可靠、不撒谎的结果。**
>
> Argos 是一个独立的通用智能体。它不让模型变聪明,只用确定性工程让模型执行你意图的
> 可靠性变高,并在它卡住或想撒谎时当场拦下。

本文每个论断都附实测证据(脚本/测试可复现),不是愿景陈述。

> **v4 变更(2026-06-01)**:从 v3 的「站在 Hermes 之上的多 agent 蜂群工作台」彻底转向。
> Hermes 已移除(它是同类竞品,非依赖)。单 agent loop + 确定性护城河。
>
> **v5 变更(2026-06-05)**:形态定为**单进程终端(TUI)超级智能体**(2026-06-03 删 Tauri/React
> 桌面壳)。引擎从 LangGraph 重写为**自建 framework-free CodeAct loop**(仅依赖 smolagents 做沙箱
> 执行器);彻底**模型无关**(Anthropic + OpenAI 双协议,`argos setup` 接任意模型)。能力从
> 「编码 + 检索」扩到 **Claude Code + 通用 agent 合体**:计算机控制(浏览器自动化)、原生 MCP
> 接入、按任务召回的 Skills、结构化任务契约层、**Dynamic Workflows(声明式 fan-out 编排,
> 模型无关 per-agent,子 agent 隔离+逐阶段 verify)** —— 全部遵守同一条诚实底线。所有旧
> LangChain/LangGraph/FastAPI 死栈已清除(详见 CHANGELOG 2026-06-05)。灵魂与三道防线(§2)不变。

## 1. 它解决的真实痛点

- 很多人觉得 Claude / GPT 贵,或在国内根本调不到,但又想用便宜模型(MiniMax/GLM/Kimi/
  DeepSeek)把事干成。
- 便宜模型裸用的真实问题不只是「不够聪明」,更要命的是**它会对你撒谎**——做错了也说
  「已完成/已修复」,让你拿到坏结果还以为成了。
- **Argos 的命题**:不靠换更强的模型,靠一层确定性工程,让便宜模型的产出从「大概率对、
  还会骗你」变成「确定对、不骗你」。这是**可及性(access)+ 诚实**,不是性价比。

**对手不是 Claude**(那扇门对用不起的人本就关着),**对手是「便宜模型裸用的那个烂结果」。**

---

## 2. 核心机制:三道确定性防线(全部实测/测试锁死)

### ① verify 硬门禁(让「完成」绕不过验证)
agent 声称「完成」时,Argos 不信文字,强制跑一条可机检的验证命令(测试/编译/lint),
**退出码非 0 就把真实报错 bounce 回去逼它继续修**。模型能对人撒谎,但绕不过 `exit_code`。

### ② escalation(卡住时诚实喊人,而非假装)
反复修仍过不了(达上限)→ agent **诚实升级**:「我没搞定,试过 X,需要你指路」,
绝不允许它以「完成」收场。对应你要的「不知道升级路由」。

### ③ 防作弊隔离 / 篡改可见(测谎仪不能被贿赂)
- 沙盒模式:验证物放 agent 写不到的隔离区,它改不了评判自己的测试。
- 项目模式(用户自己的项目):测试就在项目里改得到,改用**篡改可见**——agent 动了
  测试文件,run 结束显著警告用户。**实测堵住过:agent 真把「不可能通过」的测试改成
  `pass` 来作弊。**

加上**契约层**(仅结构化工程任务注入「必检约定 checklist」,逼便宜模型把字段/类型/
命名定死,可组装)——这是唯一有 8→0 冲突实测数据的差异化资产。

## 3. 实测证据(可复现)

**护城河有效性**(agent/cost_ab.py,退出码裁决,隐藏测试隔离区):

| 模型 | 裸调一次过 | Argos 闭环交付 | verify 净提升 |
|---|---|---|---|
| MiniMax-M2.7 | 33% | 100% | +67pp |
| MiniMax-M3 | 67% | 100% | +33pp |

**端到端(懂技术用户真实场景)**:在用户自己的项目里,agent 修对一个有 bug 的函数、
跑用户自己的 pytest 验证通过(独立复核退出码 0)、没篡改测试。

**回归防线**:476 个 pytest 锁死安全边界与行为(路径牢笼/命令白名单/退出码/防作弊隔离/
篡改检测/契约分类/CodeAct 契约/浏览器与 MCP 端到端等),覆盖率 ≥80%;`argos --selftest`
不连网整机自检 + 打包 binary smoke 全绿。

---

## 4. 边界(诚实标注)

- **只对「结果可被验证的事」成立**:代码/schema/配置/有明确对错的结构化任务。护城河是
  verify,没有客观对错的事(开放分析/战略/创作)verify 帮不上,便宜模型给不了强模型的
  聪明。**宣称「任何事都能干成」本身就是献媚式撒谎。**
- 契约层只对结构化工程任务有效;对开放式写作有害(实测 15>10),故自动不注入。

## 5. 架构(v5,单进程 TUI · framework-free)

```
单个 Python 进程(Textual TUI)
  └─ 自建 CodeAct 引擎(core/loop.py,framework-free,仅 smolagents 做沙箱执行器)
       ├─ 四阶段不可跳:plan → act(抽 ```python 代码块→沙箱执行→回灌)→ verify → report
       ├─ 任意模型(core/protocols.py:Anthropic + OpenAI 双协议;argos setup 接入)
       ├─ 工具(15 个,注入沙箱命名空间):
       │    · 文件:read/write/edit/search_file        · 计划:update_plan(真 TODO 拆解)
       │    · shell:run_command(白名单+Seatbelt+路径牢笼)· 验证:propose_verify(独立硬门禁)
       │    · 联网:web_search / web_extract            · 计算机控制:browser_*(浏览器自动化 ×5)
       │    · 外部工具:mcp_call(原生 MCP,stdio JSON-RPC)
       ├─ verify 硬门禁 + escalation(harness)        ├─ 审批闸(broker:egress+审批+HMAC 回执)
       ├─ Skills 召回 + 契约层(按 goal 注入系统提示)  └─ SQLite 记忆 + 向量/FTS5 召回 + 上下文压缩
  TUI:argos-night 主题 / Markdown 高亮 / 右侧诚实活动栏(模型·进度·工具·回执·成本)/ 启动 logo
```

模型是可替换 provider —— **不绑任何单一模型**(`argos setup` 选 provider 填 key 即可)。
打包为单个 arm64 binary(PyInstaller,`argos --selftest` 验整机装配)。

## 6. 卖点定位(M3 实测逼出的转向)

**关键发现**:模型每强一代,verify「提升交付率」的价值就缩水一半(M2.7 的 +67pp →
M3 的 +33pp,趋势向下)。所以「我们把交付率提升 N%」是**会被模型变强吃掉的流沙卖点**。

**卖点必须钉在不随模型强度缩水的三条**:
1. **确定 100% 而非 67%**:对真干活的人,「基本能用」和「确定能用」是两回事。无论模型
   多强,只要它不是 100% 可靠,verify 就是「从大概率对到确定对」的那一步。
2. **不撒谎**:强模型做错时一样会为迎合说「已完成」。verify+escalation 抓这个,与模型
   强度关系不大。
3. **便宜又可靠的可及性**:M3 又便宜又强,M3+Argos = 花很少钱拿可靠结果 = 平权。

## 7. 目标用户与现状

- **先做:懂技术的个人 / 独立开发者**(会写代码、会跑命令,只是不想付强模型高价)。
  他们能自己写 verify_cmd,护城河对他们**现在就可用**。✅ 核心路径端到端打通。
- **后做:半技术 / 普通人**(不会写验证命令)→ 需要 Argos 自动推断/生成验证方式。

**待办**:context engineering(长任务 compaction)、PyInstaller 打包(分发)、普通人的
自动验证生成、强模型成本对照(用户判断为无关问题,暂不做)。
